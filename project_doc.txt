1. Project Proposal

1.1 Project Title

SmartDocs AI — AI-Powered Document Converter and Editor

1.2 Introduction

SmartDocs AI is a web-based platform that helps users convert, edit, summarize, and improve documents using artificial intelligence. The website is designed for students, teachers, freelancers, researchers, and office users who regularly work with PDFs, images, text files, resumes, reports, and academic documents.

1.3 Problem Statement

Many users need multiple websites for PDF conversion, text extraction, grammar correction, summarization, rewriting, and document editing. Most tools are paid, limited, difficult to use, or require login. This creates inconvenience and wastes time. SmartDocs AI solves this problem by providing multiple AI-powered document tools in one website.

1.4 Objectives

Create a simple and fast AI document utility website.

Allow users to summarize, rewrite, and improve content easily.

Provide document-related tools without requiring a database in the MVP.

Make the website responsive and easy to use on mobile and desktop.

Prepare the project for public deployment on platforms such as Vercel or Render.

1.5 Scope

PDF to text conversion

Image to text extraction using OCR

AI text summarization

AI grammar correction

AI content rewriting

Project report and document generation

Downloadable output

1.6 Target Users

Students

Teachers

Researchers

Freelancers

Content writers

Office employees

Job seekers

1.7 Expected Outcome

The expected outcome is a fully functional AI document converter and editor website that can be deployed publicly, demonstrated in college, and used as a portfolio project.

1.8 Future Scope

2. SRS — Software Requirement Specification

2.1 Purpose

This SRS defines the functional and non-functional requirements of SmartDocs AI. The purpose of the software is to help users convert, extract, summarize, rewrite, and improve document content using AI.

2.2 Product Perspective

The system will work as an independent web application. It will not require a database in the initial version. Processing will happen through frontend utilities, backend APIs, file processing modules, and AI API integration.

2.3 User Classes

Normal User: Uploads files, enters text, uses AI tools, and downloads output.

Developer/Admin: Maintains APIs, improves UI, manages deployment, and updates AI prompts.

2.4 Functional Requirements

ID

Requirement

Description

FR1

File Upload

The system shall allow users to upload PDF, image, text, and notebook files.

FR2

Text Extraction

The system shall extract text from uploaded documents or images.

FR3

AI Summarization

The system shall summarize long text into short and clear points.

FR4

Grammar Improvement

The system shall improve grammar, spelling, and sentence clarity.

FR5

Content Rewriting

The system shall rewrite content in simple, formal, professional, or humanized tone.

FR6

Report Generation

The system shall generate project reports, proposals, and document drafts using AI.

FR7

Output Download

The system shall allow users to copy or download generated output.

FR8

Error Handling

The system shall show proper error messages if upload or processing fails.

2.5 Non-Functional Requirements

Performance: The website should respond quickly for normal text and file sizes.

Usability: The UI should be clean, simple, and mobile-friendly.

Security: Uploaded files should not be permanently stored in the MVP.

Reliability: The system should handle invalid files and API failures safely.

Scalability: The project should support future login, storage, and payment features.

2.6 Software Requirements

Modern browser such as Chrome, Edge, Safari, or Firefox

Node.js or Python environment

VS Code

Git and GitHub

AI API key

Deployment platform such as Vercel, Netlify, Render, or Railway

2.7 Constraints and Assumptions

Users must have internet access.

AI response depends on API availability.

File size limits will be applied.

No database will be used in the first version.

Large files may take more processing time.

4.1 Architecture Overview

SmartDocs AI follows a simple client-server architecture. The frontend collects user input, the backend processes the request, the AI service generates output, and the final result is displayed to the user.

4.2 Main Components

Backend API Server

AI API Service

Extract text from files

4.6 API Endpoints

Endpoint

Purpose

POST /api/extract-text

Extracts text from uploaded file.

POST /api/summarize

Generates summary from input text.

POST /api/rewrite

Rewrites text based on selected tone.

POST /api/grammar-fix

Improves grammar and readability.

POST /api/generate-report

Generates project documentation content.

4.7 Deployment Design

The frontend can be deployed on Vercel or Netlify. The backend can be deployed on Render, Railway, or Vercel serverless functions. API keys will be stored in environment variables.

5.4 Function Design

validateFile(file): Checks file type and size.

extractText(file): Extracts text based on file type.

generateSummary(text): Sends summarization prompt to AI API.

rewriteText(text, tone): Rewrites content based on selected tone.

fixGrammar(text): Corrects grammar and improves clarity.

generateReport(topic): Generates project documentation draft.

downloadText(output): Creates downloadable text file.
